# DJ Board Mockup Comparison — shotgun 4 variants

텍스트 목업(ASCII + CSS 의도 서술). 실제 rendered screenshot은 phase6에서 dev 서버 기반 before_after 스냅.

---

## v1 — "Spotify-like sidebar playlist"

**콘셉트**: 좌측 Now-Playing 대형 카드 (60% 너비) + 우측 세로 리스트.

```
┌───────────────────────────────┬──────────────────────────┐
│  ▶ Now Playing                │  Up Next (12곡)          │
│  ┌────────┐                   │  ───────────────────     │
│  │ 썸네일 │  IU — Love Poem   │  ⋮ 🎵 Dynamite  • 가온  │
│  │ 480x270│  IU Official      │  ⋮ 🎵 Next Level • 민준 │
│  └────────┘  ─────────────    │  ⋮ 🎵 Spring Day • 지호 │
│  │ ████▁▁▁▁ 2:14 / 3:42       │  …                       │
│  │ [⏭ 다음으로]                │                          │
│  └───────────────────────────  │  ──────────────────      │
│                                │  [+ 곡 신청]              │
└───────────────────────────────┴──────────────────────────┘
```

장: Now-Playing 시각적 존재감. 영상 embed 확장(R2) 시 자연스러움.
단: 큐 리스트가 좁아 가독성 떨어짐. 학생 뷰와 DJ 뷰 모두 동일 width라 공간 비효율. 초기 MVP엔 과함(embed 없으니 썸네일만으로 60% 채우기 썰렁).

## v2 — "Vertical list with pinned now-playing" ⭐ **채택**

**콘셉트**: 단일 세로 column. 최상단 Now-Playing 행이 크게, 나머지는 표준 큐 행.

```
┌─────────────────────────────────────────────────────────┐
│  🎧 DJ 큐 · 12곡 · DJ: 가온                              │
│─────────────────────────────────────────────────────────│
│  ┌─────────────────────────────────────────────────┐    │
│  │ ▶ NOW PLAYING                           ⏭       │    │
│  │ [120x68 썸네일]  IU — Love Poem              │    │
│  │                  IU Official · 가온님 신청      │    │
│  └─────────────────────────────────────────────────┘    │
│  ─────────────────────────────────────────────────     │
│  ⋮⋮ [썸네일] Dynamite             BTS  · 민준님 · ✓승인 │
│  ⋮⋮ [썸네일] Spring Day          BTS  · 지호님 · ⏳대기 │
│  ⋮⋮ [썸네일] Next Level         aespa · 지호님 · ⏳대기 │
│     …                                                    │
│─────────────────────────────────────────────────────────│
│                                       [+ 곡 신청]        │
└─────────────────────────────────────────────────────────┘
```

장: 정보 계층 명확 (§2 Design Brief 그대로). 모든 뷰포트(모바일/데스크탑)에서 동일 레이아웃. 비-DJ 학생에겐 ⋮⋮ 드래그 핸들만 숨기면 되어 diff 최소.
단: Now-Playing 영역이 v1보다 작음 — 하지만 MVP엔 썸네일 embed만 있으니 오히려 적절.

**채택 이유**: Design Brief §1 정보 계층 1→2→3 흐름과 정확히 매치. 접근성(§4) 측면에서 단일 column이 스크린리더 동선 단순. 드래그 범위도 큼(column 전체).

## v3 — "Chat-style queue (newest at bottom)"

**콘셉트**: 메신저처럼 오래된 곡이 위, 새 곡이 아래. Now-Playing은 최상단 "헤더" 스타일.

```
┌─────────────────────────────────────────────────────────┐
│  ▶ 지금 재생: Love Poem — IU  ⏭                          │
│─────────────────────────────────────────────────────────│
│  [2:30] Dynamite — BTS (가온)               ✓ 승인      │
│  [2:15] Spring Day (민준)                    ⏳ 대기    │
│  [1:45] Next Level (지호)                    ⏳ 대기    │
│  …                                                       │
│  ─────────────────── 최근 신청 ──────────                │
│  [방금] Blueming (가온)                      ⏳ 대기     │
│─────────────────────────────────────────────────────────│
│                                       [+ 곡 신청]        │
└─────────────────────────────────────────────────────────┘
```

장: 방금 신청한 곡이 눈에 잘 띔 (학생 체감).
단: "다음에 재생될 순서" 감각이 뒤바뀜 (보통 위가 우선). DJ가 재정렬할 때 방향 혼란. phase0 "A1 Spotify-style"에 어긋남.

## v4 — "Two-column: 대기/승인"

**콘셉트**: 좌측 "신청 대기" · 우측 "승인된 큐" 2-column. 승인=좌→우 이동.

```
┌────────────────────────┬───────────────────────────────┐
│ ⏳ 신청 대기 (5곡)     │ ✓ 승인됨 큐 (7곡) - DJ만 이동  │
│ ────────────────────── │ ──────────────────────────────│
│ [썸] Blueming · 가온   │ [썸] Love Poem · IU  ▶ PLAYING │
│ [썸] Eight · 지호      │ [썸] Dynamite · BTS            │
│ …                      │ [썸] Spring Day · BTS          │
│                        │ …                              │
│ [+ 곡 신청]             │                                │
└────────────────────────┴───────────────────────────────┘
```

장: 승인/거부 워크플로우가 시각적으로 명확. 칸반 스타일.
단: 모바일에서 붕괴. DJ가 좌→우 드래그해야 하는데 세로 리스트 내 순서 변경과 의미가 섞임. 구현 복잡도↑.
