# Mockup Comparison — Portfolio Page Layout

shotgun 4 variants. UI 의 큰 골격(학생 탐색 entry-point) 결정.

## v1 — Classic 2-pane (좌측 리스트 + 우측 그리드) ✅ 선택

```
┌──────────────────────────────────────────────────┐
│  포트폴리오                                       │
├──────────┬───────────────────────────────────────┤
│ 우리 학급 │  홍길동의 작품 (12)                   │
│  🟢 1.김  │  ┌──┐ ┌──┐ ┌──┐ ┌──┐                 │
│   2.이   │  │..│ │..│ │..│ │..│                 │
│   3.박   │  └──┘ └──┘ └──┘ └──┘                 │
│   4.최   │  ┌──┐ ┌──┐ ┌──┐ ┌──┐                 │
│   ...   │  │..│ │..│ │..│ │..│                 │
└──────────┴───────────────────────────────────────┘
```

**장점**:
- 학급 학생 컨텍스트가 항상 가시 — 비교/탐색 빠름
- 사용자 명시 요구와 1:1
- Mobbin/Slack/연락처 앱 보편 패턴 → 학습 비용 0
- 모바일 stack 변환 명확 (리스트 → 상세 push)

**단점**:
- 데스크톱에서 좌측 항상 점유 (~240px)
- 학급 30+ 명이면 좌측 스크롤 길어짐 (출석번호 chip 으로 완화 가능)

## v2 — Top tab bar (가로 학생 탭 + 하단 그리드) ❌

```
┌──────────────────────────────────────────────────┐
│  포트폴리오                                       │
├──────────────────────────────────────────────────┤
│ [🟢김] [이] [박] [최] [정] [...] →               │
├──────────────────────────────────────────────────┤
│  김길동의 작품 (12)                              │
│  ┌──┐ ┌──┐ ┌──┐ ┌──┐ ┌──┐ ┌──┐                  │
│  │..│ │..│ │..│ │..│ │..│ │..│                  │
│  └──┘ └──┘ └──┘ └──┘ └──┘ └──┘                  │
└──────────────────────────────────────────────────┘
```

**장점**: 그리드 가로 공간 최대 활용
**단점**: 학급 30명 가로 탭 = 끝없는 좌우 스크롤. 작품 수 메타 표시 어려움. 사용자 의도("좌측 학생목록")와 불일치.

## v3 — 사진 카드 그리드 진입 (Pinterest-like) ❌

```
┌──────────────────────────────────────────────────┐
│  포트폴리오                                       │
├──────────────────────────────────────────────────┤
│  ┌────┐ ┌────┐ ┌────┐ ┌────┐                    │
│  │1.김│ │2.이│ │3.박│ │4.최│ ...                 │
│  │아바│ │아바│ │아바│ │아바│                     │
│  │타+ │ │타+ │ │타+ │ │타+ │                     │
│  │12개│ │8개 │ │5개 │ │0개 │                     │
│  └────┘ └────┘ └────┘ └────┘                    │
│  (학생 카드 클릭 → 별도 페이지로 이동)             │
└──────────────────────────────────────────────────┘
```

**장점**: 시각적으로 풍성, 아바타 색상 구분 가능
**단점**: 학생 → 작품 이동에 페이지 전환 1회 추가. 빠른 비교 어려움. 컨텍스트 스위칭 비용 증가.

## v4 — 3-pane (학생 + 작품 + 상세) ❌

```
┌──────┬──────────────────┬───────────────────────┐
│ 학생  │  작품 그리드      │   카드 상세           │
│ 리스트│                   │   (선택된 카드)        │
│      │  ┌──┐ ┌──┐        │   ┌──────────────┐  │
│      │  │..│ │..│        │   │  이미지       │  │
│      │  └──┘ └──┘        │   │              │  │
│      │  ┌──┐ ┌──┐        │   │  제목·본문    │  │
│      │  │..│ │..│        │   │              │  │
│      │  └──┘ └──┘        │   │  출처 보드     │  │
└──────┴──────────────────┴───────────────────────┘
```

**장점**: 빠른 카드 detail 미리보기
**단점**: ≥1280px 화면 전제. 모바일/태블릿 깨짐 심함. 카드 detail 은 기존 CardDetailModal 이 이미 처리 → 중복. 복잡성 대비 가치 낮음.
