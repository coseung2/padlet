# Variant v3 — Single-Column Stack + Sticky Code Drawer

컨셉: 태블릿(Tab S6 Lite) 우선. 교사 페이지를 **세로 1-컬럼 stack** (인박스 → 연결됨) + **우상단 고정 "코드 보기" pill 버튼** 으로 코드 섹션을 on-demand SidePanel 로 분리. 데스크탑·태블릿·모바일 모두 동일 모델. 학부모 온보딩은 **"체크리스트" stepper** (단계 완료 시 체크).

차별 요소:
- 초대 코드 섹션을 **기존 SidePanel 프리미티브 재사용** (design-system §7)
- 인박스 행은 **스와이프 액션** 지원 (태블릿 터치 UX 최적화) — 우측 스와이프 = 승인, 좌측 스와이프 = 거부
- Stepper 는 **체크리스트** (완료된 단계에 체크 아이콘)
- D+N 색은 **v1 과 동일 (신규 warning 토큰)**

### 1.2 모바일 (<768px)

동일 1-컬럼 레이아웃. 우상단 코드 버튼은 아이콘 only (🔑 + "코드" 라벨은 aria-label). SidePanel 은 모바일에서 **bottom sheet (max-height 85vh)** 로 전환 (기존 SidePanel 프리미티브 기본값).

### 1.3 스와이프 액션 (태블릿 전용)

- 행을 좌→우 스와이프 (150px 이상) → 승인 확정 전 "스와이프해서 승인" green ribbon 노출 + 500ms 내 손 떼면 승인 실행.
- 행을 우→좌 스와이프 → "거부 사유 선택" bottom sheet 오픈.
- `prefers-reduced-motion: reduce` 시 스와이프 비활성, 기본 버튼만 사용.
- 키보드 only 사용자는 버튼만 사용 (스와이프 필수 아님).

### 1.4 상태별

- empty: 인박스 섹션 = "현재 승인 대기 중인 학부모가 없습니다." illustration 없음, 13px 추가 hint "초대 코드를 학부모에게 공유해 보세요." + 바로 밑에 **CTA 겸용 코드 버튼 (secondary)** 노출.
- loading: 행 skeleton 3개.
- error: 섹션 상단 banner.
- success: 행 fade-out + toast.

## 2. 학부모 온보딩 — 체크리스트 stepper

### 2.1 공통 레이아웃

```
┌───────────────────────────────┐
│        Aura-board              │
│                               │
│   ☑ 가입                       │  ← 완료 = 체크 + 초록 체크 색 (accent 재사용)
│   ☑ 학급 코드                  │
│   ▸ 자녀 선택 (현재)            │  ← 현재 = 화살표 + accent
│   ○ 승인 대기                   │  ← upcoming = 원
│                               │
│   학부모 신청                   │  ← Title 20
│   ...                         │
└───────────────────────────────┘
```

- Stepper 는 **좌측 세로 정렬** (태블릿 가로에서 폼과 나란히 배치 가능, 모바일에서는 상단 가로 축소).
- 체크 아이콘 size 16px, 색 `--color-accent-active`.

### 2.2 P4 학생 선택 — hybrid 검색형

```
┌──────────────────────────────────────────┐
│ ☑ 가입  ☑ 코드  ▸ 자녀  ○ 대기            │
│                                          │
│ 3학년 2반 (담임 선생님)                   │
│ 25명                                     │
│ ┌──────────────────────────────────────┐ │
│ │ 🔍 이름 첫 글자 또는 번호 입력...     │ │  ← 검색 input
│ └──────────────────────────────────────┘ │
│                                          │
│ ┌────────┬────────┬────────┐            │
│ │  3-2-01│  3-2-02│  3-2-03│  3×grid    │  ← 카드 88×88 (태블릿 108×108)
│ │  김O민 │  이O준 │  박O서 │            │
│ │  ○     │  ○     │  ●     │            │
│ └────────┴────────┴────────┘            │
│ ...                                      │
│                                          │
│  [ 박O서 학생의 학부모로 신청 ]  (sticky)│  ← 선택된 이름이 CTA 에 inline

```

- 선택된 학생 이름을 CTA 텍스트에 직접 반영 (확인 효과).
- 카드는 3열 고정, 나머지 스크롤. 가상 스크롤 없음(학급당 최대 40명 이하 가정).

### 2.3 P5 Pending — 체크리스트 완료 상태

```
┌───────────────────────────────┐
│   ☑ 가입 ☑ 코드 ☑ 자녀 ▸ 대기 │
│                               │
│   승인 대기 중                 │
```

### 2.4 P6 Rejected

체크리스트 숨김. 에러 surface.

## 3. 이메일 — 공통

v1 과 동일 구조. 교사 PII 비노출.

## 4. 사용 토큰

| 용도 | 토큰 |
|---|---|
| SidePanel | 기존 `.side-panel` 프리미티브 (design-system §7) |
| 체크 아이콘 색 | `--color-accent-active` |
| D+N 3색 | v1 과 동일 (**신규 `--color-warning`**) |
| 스와이프 ribbon | 승인=`--color-accent-tinted-bg`, 거부=`#fee2e2` |
| 카드/모달/탭 | 기존 토큰 |

## 5. 컴포넌트 (신규)

- `<InviteCodeDrawer/>` — SidePanel 를 감싼 컨테이너
- `<PendingRow/>` + swipe gesture (기존 dnd-kit 안 씀, 별도 touch handler)
- `<DPlusBadge/>`, `<CodeInput8/>`, `<StudentPickerCard/>`
- `<OnboardingChecklist/>` — 세로 체크리스트 stepper

## 6. 상태 전환

- SidePanel 진입 250ms ease-out (기존 토큰), reduced-motion instant.
- 스와이프 gesture = 사용자 손 추적, transition 없음 (dnd-kit 원칙).

## 7. 장점 / 단점

- **장점**: 태블릿 최적화(스와이프), 코드는 on-demand 라 인박스에 시야 집중, SidePanel 재사용으로 신규 패턴 최소화.
- **단점**: "코드 보기" 1-클릭 비용(자주 쓰는 교사에게 부담), 스와이프 학습 곡선, 키보드 사용자는 기능 동등하지만 시각적 친화도 ↓, 체크리스트 stepper 가 기존 디자인 시스템에 없어 신규 패턴 추가 필요.
