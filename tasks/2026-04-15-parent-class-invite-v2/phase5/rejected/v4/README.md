# Variant v4 — Dashboard Cards (Triage-Focused)

컨셉: 교사 페이지 상단에 **3-card summary dashboard** (빨강 `[D+6↑: N]` / 노랑 `[D+3~5: N]` / 회색 `[대기: N]`) 을 두고, 카드 클릭 = 인박스 필터 적용. 아래는 `FilterBar` 와 동일 역할. 인박스는 동일하지만 **"긴급" 행은 좌측 3px 빨간 accent bar + 상단 고정**. 초대 코드는 v1 처럼 우측 컬럼에 노출. 학부모 온보딩은 **minimal (progress 숨김, 단일 카드 연속 이동)** — Karpathy Simplicity 극단.

차별 요소:
- 대시보드 상단 3-card 요약 (triage)
- 학부모 온보딩 **stepper 없음** (단일 카드 시퀀스)
- D+N 색은 **v1 과 동일 (신규 `--color-warning`)**
- 학부모 P4 는 **자동완성 검색 입력** + 드롭다운 — 카드 그리드/리스트 없음

- 상단 3-card 는 `--color-danger` tinted(#fee2e2) / warning tinted(#fef3c7) / `--color-surface-alt` 배경.
- 3-card 클릭 시 아래 인박스의 FilterBar 가 해당 필터 활성화.
- "긴급" 행 (D+6↑) 은 좌측 3px 세로 바 `--color-danger` + 배경 `rgba(198,40,40,0.04)`.
- Grid: `minmax(0, 8fr) minmax(320px, 2fr)` — 인박스가 훨씬 넓음.

### 1.2 태블릿

3-card 는 가로 스크롤 가능 (touch friendly, 카드별 min-width 180px). 인박스·코드·연결됨 세로 스택.

### 1.3 모바일

3-card 를 **단일 compact summary pill** (`긴급 3 · 주의 5 · 대기 12`) 로 축소.

### 1.4 상태별

- empty: 3-card 모두 0건. 중앙 "아직 대기 중인 학부모가 없습니다" + 초대 코드 섹션만 부각.
- loading: 3-card skeleton + 행 skeleton.
- error: 3-card 영역에 banner.
- success: 승인 시 해당 triage 카드 카운트가 1 감소 (숫자 150ms ease count-down).

## 2. 학부모 온보딩 — stepper 없음 (minimal)

### 2.1 설계 의도

brief §2.2 는 "단일 핵심 행동 1개 원칙" 만 요구. v4 는 여기서 **stepper UI 자체를 제거** 하고, 헤더에 매우 작은 텍스트 `1 / 4` 만 표시. 학부모는 모바일에서도 한 화면 = 한 태스크만 집중.

## 3. 이메일 — 공통

v1 과 동일 구조.

## 4. 사용 토큰

| 용도 | 토큰 |
|---|---|
| triage 카드 빨강 | `--color-danger` + tinted `#fee2e2` + 1px border `rgba(198,40,40,0.15)` |
| triage 카드 노랑 | **신규 `--color-warning`** `#f59e0b` + tinted `#fef3c7` |
| triage 카드 회색 | `--color-surface-alt` + `--color-text-muted` |
| 긴급 행 좌측 바 | `--color-danger` 3px |
| 인박스/코드/연결됨 카드 | 기존 card 패턴 |
| 학부모 온보딩 | 기존 토큰만, Display/Title/Body 매핑 |

## 5. 컴포넌트 (신규)

- `<TriageSummaryCards count={...}/>` — 3-card 클릭 시 인박스 필터 toggle
- `<PendingRow/>` (좌측 3px bar variant)
- `<DPlusBadge/>`, `<CodeInput8/>`
- `<StudentComboBox/>` — ARIA combobox 패턴 (grid/list 대체)

## 6. 상태 전환

- triage 카드 → 필터 적용: 인박스 행 crossfade 150ms.
- 카운트 변화: `CountUp` 숫자 150ms.
- 학부모 페이지 전환: 기본 router.push (animation 없음, instant).

## 7. 장점 / 단점

- **장점**: 트리아지 UX 가 교사의 D+N 긴급 판단을 시각적으로 즉시 지원, "긴급 건수" 대시보드가 심리적 허들 완화, 학부모 온보딩이 극도로 단순 (Karpathy §2 준수).
- **단점**: 3-card + inbox + code + linked = 정보 밀도 높아져 태블릿 세로에서 스크롤 길어짐, 학부모 stepper 제거로 "몇 단계 남았지?" 피드백 감소, StudentComboBox 는 그리드 탐색보다 **이름을 모르는 학부모** 에게 불친절 (자녀 이름을 정확히 몰라도 찾아야 하는 부모 케이스 고려).
