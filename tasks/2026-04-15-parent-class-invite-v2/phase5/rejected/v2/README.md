# Variant v2 — Tabs (Split Inbox / Code / Linked)

컨셉: 교사 페이지를 **단일 컬럼 + 상단 탭 3개** 로 구성. 인박스 우선순위를 "탭 기본값" 으로 구현. 정보 밀도는 낮지만 태블릿·모바일에서 반응형 브레이크포인트 없이 동일 UX. 학부모 온보딩은 **progress bar** 스타일 (dot stepper 대신).

차별 요소:
- 교사 페이지 = 탭 UI (기본 탭 = 인박스)
- D+N 노랑은 **기존 `--color-accent-tinted-*` 토큰 재사용** — 신규 토큰 0개 (tokens_patch.json = {})
- Stepper 는 **horizontal bar** (0% → 25% → 50% → 75% → 100%)
- P4 학생 선택 = **리스트형 (1열)** — 카드 그리드 대신

### 1.3 태블릿 / 모바일

동일 탭 UI. 탭 라벨만 아이콘 + 카운트로 축소 (`📥 12` `🔑` `✓ 18`), aria-label 에 full text.

### 1.4 상태별

- empty (인박스): 탭 안 중앙 "아직 대기 중인 신청이 없습니다" + sub.
- loading: 행 skeleton × 5.
- error: 탭 콘텐츠 영역에 red-tinted banner.
- success: 행 fade-out + toast.

## 3. 이메일 — 공통

v1 과 동일 구조. 단, Aura-board 헤더가 흰 배경 + 1px 보더로 탭 UI 의 "정적" 느낌과 일치.

## 4. 사용 토큰

| 용도 | 토큰 |
|---|---|
| 탭 active underline | `--color-accent` 2px |
| 탭 inactive | `--color-text-muted` |
| 탭 hover | `--color-accent-tinted-bg` |
| D+N 배지 3색 | **기존 토큰만**: 회색=`--color-surface-alt`/text-muted · 노랑=`--color-accent-tinted-bg`/`--color-accent-tinted-text` · 빨강=`--color-danger`/tinted bg (hardcoded `rgba(198,40,40,0.08)`) |
| Progress bar | `--color-accent` / `--color-border` |

> 주의: v2 는 "노랑 = accent-tinted 재사용" 이라 brief §5.4 권고(신규 warning 토큰)와 의도적으로 충돌. tokens_patch.json = `{}`.

## 6. 상태 전환

- 탭 전환: 콘텐츠 crossfade 150ms (reduced-motion instant).
- 나머지는 v1 과 동일.

## 7. 장점 / 단점

- **장점**: 모든 해상도 동일 UX, 인박스 기본 진입, 토큰 0개 신규 (design-system 영향 없음), 탭 URL 싱크로 딥링크 지원.
- **단점**: 인박스 — 초대 코드 간 전환 비용(탭 클릭 1회) 발생, 교사가 "인박스+코드" 동시 참조 시 불편, D+N 노랑이 accent 와 의미 혼동 가능 (accent = CTA 색).
