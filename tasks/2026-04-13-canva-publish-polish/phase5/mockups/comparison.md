## 사용자 지정 선정: v3

**근거 (사용자 지시):**
1. 학급 맥락에서 작성자 존재감·귀속을 시각적으로 강조하는 chip 형태 선호.
2. 이름 영역이 시각적으로 분리되어 "누가" 정보 스캔성 최상.
3. 후속 task 에서 작성자 프로필 열기 등 interactive 승격 시 자연스러운 경로.
4. chip/pill 기반 표시가 제품의 미적 톤과 더 잘 맞는다는 사용자 판단.

## 기각

- **v1 Minimal**: 작성자 존재감 약함, 학급 맥락 스캔성 부족.
- **v2 Weighted**: 이름 가독성은 좋으나 chip 대비 시각적 구분감 낮음.
- **v4 Stacked**: 카드 높이 +40px — 태블릿 4 카드 배치 시 스크롤 비용 증가.

## v3 의 알려진 트레이드오프 (감수)

- chip 이 시각적으로 클릭 유도 신호를 주지만 본 task 에서 `pointer-events: none` 유지. 후속 task 에서 프로필 열기 기능 합류 시 `<button>` 으로 승격해 해소.
- 디자인 시스템에 tinted bg / pill radius 토큰이 없으면 task-local CSS 값 인라인 (phase6 reviewer 판정).
