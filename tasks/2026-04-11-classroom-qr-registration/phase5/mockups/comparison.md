# Design Variants Comparison — classroom-qr-registration

## v1: 탭 기반 (Tab Navigation)
학급 목록과 상세를 탭으로 전환. `/classroom` 단일 페이지에 탭 2개.
- **장점**: 페이지 이동 없이 빠른 전환
- **단점**: 학급 여러 개일 때 탭이 복잡, URL 공유 불가, 기존 앱 패턴(별도 페이지)과 불일치

## v2: 페이지 분리 + 카드 그리드 (SELECTED)
기존 Dashboard 패턴과 동일. `/classroom` = 학급 카드 그리드, `/classroom/[id]` = 학급 상세 페이지.
- **장점**: 기존 Dashboard/Board 패턴과 완전 일치, URL 공유 가능, 정보 계층 명확
- **단점**: 페이지 이동 필요 (미미한 단점)

## v3: 사이드바 + 패널
좌측 사이드바에 학급 리스트, 우측에 상세 패널. Gmail/Notion 스타일.
- **장점**: 학급 간 빠른 전환
- **단점**: 모바일 대응 복잡, 기존 앱에 사이드바 패턴 없음, 구현 복잡도 높음

## v4: 아코디언 (Accordion)
학급 목록에서 학급 클릭 → 인라인으로 학생 명단 펼침.
- **장점**: 전체 조망 가능
- **단점**: 학생 30명이면 스크롤 과다, QR 출력 UX가 어색, 정보 밀도 과다

## 결정
**v2 (페이지 분리 + 카드 그리드)** 선택. 사유:
1. 기존 앱의 Dashboard → Board 패턴과 1:1 대응 (학습 비용 0)
2. 디자인 시스템 토큰 재사용 100% (신규 토큰 불필요)
3. 학생 대시보드도 동일 카드 그리드 패턴으로 통일
4. 반응형 이미 검증된 구조
