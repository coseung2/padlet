# Design Spec — card-file-attachment

(phase4 design_planner + phase5 designer + phase6 design_reviewer 결과 압축. 이번 feature는 "기존 모달 패턴에 버튼 1개 + 카드 뷰에 분기 2개 추가"로 디자인 탐색 공간이 좁고, 단일안이 명확하다.)

## 1. AddCardModal — 파일 버튼 섹션

### 첨부 버튼 바 (기존)
```
[🖼️ 이미지] [🔗 링크] [🎬 동영상] [🎨 내 라이브러리]
```

### 개선 (신규)
```
[🖼️ 이미지] [🔗 링크] [🎬 동영상] [📎 파일] [🎨 내 라이브러리]
```

- 버튼은 기존 `.modal-attach-btn` 스타일 그대로
- 순서: 유사도 그룹화 → 이미지/링크/동영상/파일(콘텐츠) → 라이브러리(액션)
- 활성 상태 `.modal-attach-btn-active` 동일

### 파일 섹션 내부

비어있을 때 (드롭존):
```
┌─────────────────────────────┐
│   📎                         │
│   클릭 또는 파일을 드래그    │
│   PDF · Word · Excel · …     │
│   (최대 50MB)                │
└─────────────────────────────┘
```

파일 선택 후 (프리뷰):
```
┌─────────────────────────────┐
│ 🗎  내보고서.pdf              │
│     2.3 MB · PDF             │
│                   [제거]     │
└─────────────────────────────┘
```

## 2. 카드 뷰 — 파일 렌더 분기

### 2-1. PDF 데스크톱 (< 10MB)
```
┌─ card ─────────────────────┐
│ [iframe 인라인 PDF 뷰어]    │
│ (aspect-ratio 4/3)          │
│                             │
│ 제목                        │
│ 내용                        │
└─────────────────────────────┘
```
- iframe `src="{url}#view=FitH"` — 폭 맞춤 렌더
- min-height 360px, 카드 폭(240~480px)에 따라 비율 조정

### 2-2. PDF 데스크톱 (≥ 10MB)
상단에 경고 배너 추가:
```
┌─────────────────────────────┐
│ ⚠️ 파일이 커서 로딩이 느릴 수 │
│    있어요 (48.2 MB)          │
│ ┌─────────────────────────┐ │
│ │   [펼쳐서 보기]            │ │
│ └─────────────────────────┘ │
└─────────────────────────────┘
```
- `<details>` 기본 접힘. 클릭 시 iframe 마운트

### 2-3. PDF 모바일 (iOS/Android UA)
비-PDF 파일 카드와 동일 렌더로 fallback:
```
┌─────────────────────────────┐
│ 🗎  내보고서.pdf              │
│     2.3 MB                  │
│     [📥 다운로드]           │
└─────────────────────────────┘
```
- `<a download>` 또는 새 탭 열기

### 2-4. 비-PDF 문서
```
┌─────────────────────────────┐
│ 📄  기획서.docx              │
│     1.2 MB                  │
│     [📥 다운로드]           │
└─────────────────────────────┘
```
- 아이콘 크기 32px, 이름 14px 볼드, 크기 12px 뮤트
- 다운로드 버튼 `.card-attach-file-download` — `.dj-next-btn` 톤과 통일

## 3. 토큰/스타일

- 파일 카드 배경: `var(--color-surface-alt)` (이미지 없는 링크 프리뷰와 동일 계열)
- 파일 카드 border: `1px solid var(--color-border)`
- 파일 카드 radius: `var(--radius-card)` (기존과 동일)
- PDF iframe radius: 8px (동영상과 동일)
- 경고 배너: `background: rgba(255, 193, 7, 0.1); border: 1px solid rgba(255, 193, 7, 0.3);`

## 4. 접근성

- 파일 버튼 `aria-label="파일 첨부"`
- 파일 프리뷰 `aria-label="첨부된 파일: {fileName}, {formatBytes(size)}"`
- PDF iframe `title="PDF 미리보기: {fileName}"`
- 다운로드 버튼 `<a>` 태그로 렌더, `aria-label="{fileName} 다운로드"`

## 5. 반응형

- 카드 너비 < 200px: PDF iframe은 숨기고 파일 카드 렌더 (iOS와 동일 fallback)
- 모달 내 파일 섹션: 기존 이미지/동영상 섹션과 동일 패딩/간격

## 6. 탈락 변형 (기록용)

- **변형 A: 파일 + 이미지 + 동영상 단일 "미디어" 버튼으로 통합** — 기각: 학습자에게 "PDF를 미디어에서 찾아야 함"이 혼란. 기존 사용자 재학습 비용 높음.
- **변형 B: 파일을 라이브러리 드롭다운 항목으로** — 기각: 라이브러리는 학생 개인 자산 전용 의미론. 파일은 공개 업로드라 의미론이 섞임.

## 7. 검수 결과 (phase6)

- **일관성**: 기존 4개 버튼과 동일 클래스/패턴 ✅
- **정보 계층**: 파일 아이콘 → 이름(주) → 메타(부) → 액션 순서 ✅
- **접근성**: aria-label 지정 ✅
- **반응형 fallback**: iOS/소형 카드 모두 다운로드 폴백 ✅
- **디자인 검수 PASS** — phase7 구현 진행.
