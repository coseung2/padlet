# Design Shotgun — classroom-bank

각 변형은 핵심 화면 3개(학생 Wallet / 결제 POS / 권한 모달) 관점에서 비교. 실제 렌더는 phase7 구현 후 before_after에.

## v1 — Toss 영감, 단일 컬럼 + 컴팩트 모달

**콘셉트**: 모바일 우선. 학생 Wallet은 세로 긴 페이지 (잔액 → 카드 → 적금 → 거래). POS는 카트 상단 sticky, 상품 그리드 하단. 권한 모달은 섹션별 접힌 카드.

```
[/my/wallet]                 [/pay]
┌─────────────┐              ┌─────────────┐
│  잔액        │              │  총액 X원    │ sticky
│  8,500원     │              │  [결제] CTA │
├─────────────┤              ├─────────────┤
│  [카드 QR]   │              │  🧃 음료 200│
│  60s •••   │              │  🍫 초콜릿 150│
├─────────────┤              │  🍪 과자 300 │
│  📅 적금     │              │  …          │
│  10000 D-7 │              │             │
├─────────────┤              │ Cart: 음료 1│
│  최근 거래   │              │        -+  │
│  …          │              └─────────────┘
```

장: 모바일 친화, 정보 계층 선명 / 단: 데스크탑에서 좁아 보임

## v2 — 2-column 분할, 좌측 카드/우측 거래  ⭐ **채택**

**콘셉트**: Wallet은 좌측 카드+적금, 우측 거래. POS는 좌측 상품 그리드, 우측 카트. 권한 모달은 카테고리 섹션 헤더 + 체크박스 리스트.

```
[/my/wallet]                              [/pay]
┌─────────────┬──────────────┐           ┌────────────┬──────────┐
│ 잔액 8500   │  최근 거래    │           │ 상품 그리드 │  카트     │
│ ─────────── │  ────────    │           │ 🧃🍫🍪🍩  │ 총액 X원  │
│ [카드 QR]   │  -200 구매   │           │ 🍿🥤🧊...  │ [결제]   │
│  60s  ⏱    │  +1000 입금  │           │            │           │
│ ─────────── │  …           │           │            │ - + 수량  │
│ 📅 적금      │              │           │            │           │
│  10000 D-7 │              │           │            │           │
└─────────────┴──────────────┘           └────────────┴──────────┘
```

장: 정보 밀도 균형, 데스크탑/태블릿 모두 자연 / 단: 모바일에서 stack 필요

**채택 이유**: 학급 태블릿(대부분 10-11인치) 주 사용 기기라 2-col 자연스러움. 결제 POS는 실제 매장 POS UI 관습 (좌측 카탈로그 + 우측 카트).

## v3 — Wallet 카드 정면 + Flip 애니메이션

**콘셉트**: Wallet 상단에 카드 플라스틱 비주얼(표면 QR, 뒷면 잔액 정보 flip). Toss 카카오뱅크 mini의 "카드 자체가 메인" 컨셉.

```
[/my/wallet 초기]          [/my/wallet flip]
┌─────────────┐           ┌─────────────┐
│  [CARD]     │           │ 잔액 8500   │
│  QR 정면    │  flip →   │ 카드번호     │
│  60s        │           │ 최근 거래    │
└─────────────┘           └─────────────┘
```

장: 비주얼 어필, 학생 흥미 / 단: flip 애니메이션 구현 비용, 키보드 a11y 복잡, 거래 확인에 한 단계 더 필요

## v4 — 대시보드 스타일 메트릭 카드

**콘셉트**: Wallet에 메트릭 카드들 (이번달 입금, 이번달 지출, 적금 진행률 bar 등). 데이터 풍부.

```
[/my/wallet]
┌──────┬──────┬──────┐
│ 잔액  │ 이번 │ 적금 │
│ 8500 │ +2000│ D-7  │
├──────┴──────┴──────┤
│  거래 타임라인       │
└─────────────────────┘
```

장: 데이터 시각화 풍부 / 단: MVP엔 과함, 초등~중등 학생에 정보 과잉
