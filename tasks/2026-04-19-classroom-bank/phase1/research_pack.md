# Phase 1 — Research Pack (Classroom Bank)

## 조사 제한사항
자동 감독 모드라 live 캡처 생략. 공개된 제품 문서·디자인 패턴 지식 활용.

## 벤치마크 4종

### 1. **Toss (토스)** — 주력 레퍼런스
국내 학생이 친숙한 모바일 뱅킹 UX.

- **통장 화면**: 잔액 최상단 큰 숫자 + 아래 거래 내역 (최신순). 입/출금 색상 분기(빨강/초록 대신 흑/회 심플).
- **카드 QR**: 토스머니 결제 바코드/QR이 홈 화면에서 한 탭으로 접근. 타이머 시각화.
- **적금**: "n개월 뒤 만기" + "원금 + 이자" 프리뷰. 중도해지 경고 모달.
- **거래 상세**: 각 거래 탭하면 상세 모달 (시간/상대/사유/잔액).

적합성: 높음. 학생 시나리오 그대로 차용.

### 2. **카카오뱅크 미니** — 청소년 대상
14~18세 전용 모바일 뱅킹. "용돈 관리" 콘셉트.

- **카드 테마/스티커**: 학생이 카드 디자인 커스텀. 몰입감 ↑. 
- **페이 홈**: QR 전면, 잔액은 2차.
- **미션 보상**: 부모가 "심부름 완료 → 1000원" 같은 보상 흐름.

차용 포인트: **카드 디자인 프리셋 2-3개** (scope out, phase-2 후보). 미션 보상은 우리 맥락에서 "교사 입금"으로 대체 가능.

### 3. **Venmo / Cash App** (해외 청소년 앱)
- Transaction feed에 "누가 누구에게 왜" 공개 기본. 우리는 privacy 측면 학급 내만.
- 이체(peer) 중심 → 우리는 Q4-B(이체 없음)라 해당 없음.

### 4. **역할 권한 매트릭스 UX (Notion Workspace / Slack Admin)**
- 사용자 그룹 × 권한 매트릭스가 복잡해지면 "역할 카드 → 상세 모달"로 쪼개는 패턴이 표준.
- Slack: "권한 그룹" 클릭 → 슬라이드 패널에 체크박스 리스트.
- Notion: Workspace settings → "Members & groups" → 그룹 이름 클릭 → 상세.

우리 Q10 답변 UX(역할 카드 리스트 + 모달)는 **업계 표준 패턴**.

## 핵심 UX 패턴 (채택)

### P1. 잔액 최상단 + 거래 내역 최신순 (Toss)
`/my/wallet` 상단에 잔액 크게 + 카드 섹션 + 최근 거래 5~10건.

### P2. QR 자동 갱신 타이머
카드 QR 아래에 "xx초 뒤 갱신" 프로그레스 바. 60초 주기 시각화.

### P3. 적금 프리뷰 카드
"X원을 적금에 가입하면 1달 뒤 Y원을 받아요!" (원금 + 이자율 기반 예상치) — 가입 모달에서 실시간 계산.

### P4. 결제 확인 이중 게이트 (금전 실수 방지)
매점원 스캔 → 총액 + 학생명 + 현재 잔액 표시 → 명시적 "결제" 버튼. 원탭 결제 안 함.

### P5. 역할 카드 + 권한 모달 (Q10 답변)
역할 리스트 → 카드 클릭 → 모달에 체크박스 그룹.

### P6. 교사 감사 뷰
`/classroom/:id/bank` 에서 전체 거래 타임라인 (필터: 타입 / 학생 / 기간). 이상 거래 감지 (잔액 음수 등) 경고.

### P7. 낙관적 UI + 서버 검증
학생 폰에서 카드 QR 표시 후 매점원이 결제 → 학생 화면 SSE로 즉시 "-X원" 업데이트. Toss의 즉시성 매칭.

## 거절 패턴

| 패턴 | 거절 이유 |
|---|---|
| 이체 feed 공개 (Venmo) | 학급 프라이버시, Q4-B out |
| 신용카드/여신 | 디지털 교육용에 불필요 |
| 다중 통화 | Q8 단일 통화 |
| 투자/주식 상품 | 교육용에 과함 |
| 푸시 알림 | Web push 인프라 필요, scope out |

## 국내 학급화폐 교실 운영 사례

- **한국 초등교사 커뮤니티 사례**: 엑셀/수기 기반이 대부분. 디지털 전환 시 "교사가 수기 입력 → 엑셀 이자 계산" 패턴을 앱이 자동화. 기대 효과: 운영 오버헤드 80%+ 감소.
- **저축·소비·기부 3분할 지갑 개념**: 우리 MVP는 단일 통장. 향후 확장 여지.

## 이번 task에서 고정할 UX 의사결정

phase2에 전달:

5. 상단 nav: `학생명부 · 학급보드 · 학급역할 · 은행 · 매점`
