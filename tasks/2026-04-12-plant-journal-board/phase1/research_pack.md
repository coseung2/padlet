# Research Pack — plant-journal-board

## Context

교과 연계 식물 관찰 학습(초등 과학 3~4학년 "식물의 한살이" 단원 + 실과 "식물 가꾸기"). 학급 단위 교사-학생 역할이 이미 정립된 Aura-board 위에서 동작.

상위 문서(`canva project/plans/plant-journal-roadmap.md`, `plant-catalog.md`)는 본 워크트리에 미반영 상태이므로 phase0 `request.json`과 오케스트레이터 프롬프트의 PJ-1~6 스코프를 1차 근거로 한다.

## Benchmark 1 — Seesaw "Journals"

관찰 포인트:
- 학생별 개인 저널(1인당 포스트 시리즈), 교사는 반 전체 뷰
- 이미지 + 짧은 메모 조합, 1포스트 = 1관찰
- 피드형(세로 스크롤) → 단계/구조가 없어 "성장 과정"이 잘 드러나지 않음 (단점)
- 교사는 student grid 뷰로 제출 여부만 확인

**장점**: 저비용 업로드 흐름, 학생 입장 장벽 낮음
**단점**: 단계 구조 부재 → "다음 단계 진입"같은 자기 선언 UX가 불가능

## Benchmark 2 — Trello / Kanban "이동 칼럼"

관찰 포인트:
- 고정된 단계(To do/Doing/Done)와 카드 이동 은유가 학습 단계와 매핑 가능
- 드래그로 단계 변경 → 학생이 직접 "내 식물이 다음 단계로 갔다"를 선언

**장점**: 단계 시각화 명확, 익숙한 은유
**단점**: 가로 스크롤 + 다중 칼럼은 태블릿에서 답답함, "되돌아갈 수 없음"을 강조하기 어려움

## Benchmark 3 — 지하철 노선도 UI (Seoul Metro, Citymapper "Line view")

관찰 포인트:
- 일렬로 늘어선 노드 + 진행중 노드 하이라이트 = **성장 = 선형 진행**의 은유와 일치
- 노드를 tap하면 해당 역 상세 (여기선 관찰 포인트 + 내 사진)
- 모바일/태블릿에서 가로 스와이프에 강함, 세로 레이아웃은 카드 리스트로 대체 가능

**장점**: 한눈에 "지금 몇 단계인지"가 보임, 초등학생도 직관적, 시각 자산이 귀엽게 디자인 가능
**단점**: 단계 수가 많아지면(>7) 화면 밀집, 반응형 고민 필요

## Benchmark 4 — Notion Gallery + Timeline

관찰 포인트:
- 교사 매트릭스 뷰와 유사한 구조 (row=property, col=row)
- 썸네일 위주 밀집 정보, 원본은 모달

**장점**: 한 화면에 많은 학생 스캔 가능
**단점**: 태블릿에선 cell이 너무 작아 터치 정확도 떨어짐 → **owner+desktop only 정책과 부합** (단점이 아니라 제약)

## 패턴 선정 근거

- **학생 뷰**: 지하철 노선도 UI (Benchmark 3). "성장의 선형성" + "지금 내 위치" 의 두 축을 한 뷰로 표현.
- **관찰 기록**: Seesaw식 사진+메모 시트, 단 "단계 자기 선언"과 "사진 없음 사유" 두 확인 게이트 추가(Benchmark 2 은유 차용).
- **교사 요약**: 단계별 분포 + 정체 경고 (Notion property summary 변형)
- **교사 매트릭스**: Notion Gallery grid, 단 썸네일만. 원본은 모달. desktop 전용.

## 리스크 (제품/UX)

1. 단계 수(현재 10) × 학생 수(20~30) = 최대 300 셀 — 렌더/네트워크 부담
2. 학생이 단계를 역행(되돌리기) 요구 → MVP에서는 **최신 단계만 추가 가능**, 수정/삭제는 소유자만, 단계 되돌리기 OUT
3. 사진 수 상한(10/단계) 초과 시도 → 서버 측 enforce 필수
4. iPad 9세대 iOS Safari의 구 Web API — 최신 CSS/SVG 기능 사용 시 폴백 필요
