## 수동 검증 체크리스트 (phase3 검수에서 재현)

### SSE 스트림 델타 동작
- [ ] 호스트가 퀴즈 생성 → 참가 코드 표시
- [ ] 학생 2명이 동일 코드로 참가 → 호스트 뷰에 플레이어 목록 즉시 업데이트
- [ ] 호스트가 "시작" → 학생에게 첫 문제 도착
- [ ] 학생 1이 답변 → 호스트 뷰 분포 차트 업데이트 (count 변화)
- [ ] 학생 2가 답변 → 분포 차트 재업데이트
- [ ] 호스트가 "다음 문제" → 분포 리셋 + 새 문제 표시
- [ ] 호스트가 "종료" → 리더보드 표시

### 점수 업데이트 버그 수정 검증 (새 동작)
- [ ] 두 플레이어가 동시에 정답 제출 → 둘 다 점수 +N
- [ ] **플레이어 수 변화 없이** 점수만 증가해도 호스트 뷰 플레이어 목록에 점수 업데이트 반영 확인
  (Before: count 기반 비교라 score-only 업데이트 무시됐음)

### 스트림 취소 (cancel) 누수 방지
- [ ] 호스트 브라우저 탭 닫기 → 서버 로그에서 해당 quiz의 [SSE poll] 이 다음 tick에서 중단됨 확인
- [ ] 호스트 페이지 새로고침 → 기존 SSE 종료 + 새 SSE 시작 (중복 구독 없음)
